REST API Endpoint

  • Home:

    The home page of the application.

    1) http://127.0.0.1:8000 OR 2) http://127.0.0.1:8000/api

  • Create Protein:

    Add a new protein to the database.

    Example: POST http://127.0.0.1:8000/api/protein/

    Example data

    { "protein_id": "TESTP", "sequence": "dfvdffd", "taxonomy": { "taxa_id": "testtax32", "clade": "D", "genus": "fgdgfbgd", "species": "dfdfgvbg" }, "length": 23, "domains": [{ "pfam_id": { "domain_id": "TESTP32", "domain_description": "PeptidaseC13family" }, "description": "Peptidase C13 legumain", "start": 40, "stop": 94 }] }
  • Protein Detail:

    Get details of a protein by protein ID.

    Example: GET http://127.0.0.1:8000/api/protein/A0A016S8J7

  • Pfam Detail:

    Get details of a Pfam domain by domain ID.

    Example: GET http://127.0.0.1:8000/api/pfam/PF00360

  • Proteins by Taxa:

    Get proteins belonging to a specific taxa ID.

    Example: GET http://127.0.0.1:8000/api/proteins/55661

  • Pfams by Taxa:

    Get Pfam domains belonging to a specific taxa ID.

    Example: GET http://127.0.0.1:8000/api/pfams/55661

  • Coverage by Protein:

    Get coverage information for a protein by protein ID.

    Example: GET http://127.0.0.1:8000/api/coverage/A0A016S8J7

Getting Started with the Django Project

  1. Change into the project directory:

    cd bioscience

  2. Create a virtual environment:

    python -m venv env

  3. Activate the virtual environment:

    source env/bin/activate (for Unix/Mac) ./env/Scripts/activate (for Windows)

  4. Install the project dependencies:

    pip install -r requirements.txt

  5. Run database migrations:

    python manage.py migrate

  6. Populate database using CSV files:

    python manage.py populate_db

  7. Start the development server:

    python manage.py runserver
